## コンポーネント一覧

| コンポーネント | 技術 | ポート | 役割 |
|-------------|------|--------|------|
| フロントエンド | Next.js | 3000 | Web UI、ユーザーインターフェース |
| APIサーバー | Go | 8000 | ビジネスロジック、API提供 |
| データベース | PostgreSQL + pgvector | 5432 | データ保存、ベクトル検索 |
| LLMエンジン（Ollama版） | Ollama | 11434 | LLM推論 |
| LLMエンジン（vLLM版） | vLLM | 8080, 8081 | LLM推論（Chat + Embedding） |
| 文字起こし（オプション） | Whisper (pywhispercpp) | - | 音声→テキスト変換 |
| 物体検出（オプション） | YOLOv8 (ultralytics) | - | 画像内物体の検出 |

## データフロー

### AIチャット

```
ブラウザ → フロントエンド(:3000) → API(:8000) → Ollama/vLLM → 応答
                                      ↕
                                PostgreSQL (会話履歴保存)
```

### RAG（検索拡張生成）

```
1. 文書アップロード:
   ブラウザ → API(:8000) → テキスト抽出 → ベクトル化 → pgvector に保存

2. 質問応答:
   ブラウザ → API(:8000) → pgvector で類似検索 → 関連文書 + 質問を LLM に送信 → 応答
```

### 文字起こし

```
ブラウザ → API(:8000) → FFmpeg(音声変換) → Whisper(音声認識) → テキスト結果
```

### 物体検出

```
ブラウザ → API(:8000) → YOLOv8(推論) → 検出結果(JSON + 画像)
```

## デプロイ構成

### 構成パターン A: シングルサーバー（標準）

```
┌─────────────────────────────┐
│         1台のサーバー         │
│                             │
│  フロントエンド + API + DB   │
│  + Ollama/vLLM              │
│                             │
│  推奨: 16GB RAM, 8コアCPU   │
└─────────────────────────────┘
```

最もシンプルな構成。中小規模（〜50ユーザー）に適しています。

### 構成パターン B: Docker Compose

```
┌─────────────────────────────┐
│        Docker ホスト          │
│                             │
│  ┌─────┐ ┌───┐ ┌────────┐  │
│  │ app │ │api│ │postgres│  │
│  └─────┘ └───┘ └────────┘  │
│         + Ollama (ホスト側)  │
└─────────────────────────────┘
```

コンテナ管理によるデプロイ。環境の再現性が高いです。

### 構成パターン C: 分離構成（大規模）

```
┌──────────┐   ┌──────────┐   ┌──────────┐
│ Webサーバー│   │ APIサーバー│   │ DBサーバー│
│ + Nginx  │──▶│ + LLM    │──▶│PostgreSQL│
│          │   │ + GPU    │   │          │
└──────────┘   └──────────┘   └──────────┘
```

大規模環境（100ユーザー以上）向け。各コンポーネントを別サーバーに配置。
