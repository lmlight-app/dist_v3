## 2. データ管理

| No. | チェック項目 | 回答 |
|-----|------------|------|
| 2.1 | データの保管場所 | **お客様のサーバー内のみ** |
| 2.2 | データが外部（クラウド等）に送信されるか | **いいえ** — 完全オンプレミス動作 |
| 2.3 | チャットデータの保管先 | お客様環境内のPostgreSQLデータベース |
| 2.4 | アップロードファイルの保管先 | お客様環境内のローカルストレージ |
| 2.5 | RAGベクトルデータの保管先 | お客様環境内のPostgreSQL + pgvector |
| 2.6 | テレメトリ・利用統計の送信 | **なし** |
| 2.7 | ライセンス認証の通信 | **なし** — Hardware UUIDベースのオフライン認証 |
| 2.8 | データの暗号化（保存時） | PostgreSQLのTDE設定による（お客様設定） |
| 2.9 | データの暗号化（通信時） | HTTPS対応可（Nginx設定による） |
| 2.10 | データバックアップ | お客様にて実施（pg_dumpによるDBバックアップ） |

## 3. 認証・アクセス制御

| No. | チェック項目 | 回答 |
|-----|------------|------|
| 3.1 | ユーザー認証方式 | ID（メールアドレス）+ パスワード |
| 3.2 | パスワードのハッシュ化 | bcrypt によるハッシュ化 |
| 3.3 | 多要素認証（MFA） | 非対応（オンプレミスLAN内利用を想定） |
| 3.4 | セッション管理 | JWT + Cookie |
| 3.5 | ロールベースアクセス制御 | **あり** — ADMIN / SUPER / USER の3段階 |
| 3.6 | 最小権限の原則 | USERは自分のデータのみアクセス可能 |
| 3.7 | 管理者アカウントの保護 | 最後のADMINは削除・降格不可 |
| 3.8 | デフォルトパスワードの変更 | 初回ログイン後、パスワード変更を必須として案内 |
